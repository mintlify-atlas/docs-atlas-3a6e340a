title: "EfficientNet"
description: "Compound-scaled CNN family with MBConv blocks. timm includes EfficientNet B0–B8, EfficientNet-V2 S/M/L/XL, MixNet, MNasNet, and more."
---

EfficientNet was introduced in [EfficientNet: Rethinking Model Scaling for Convolutional Neural Networks](https://arxiv.org/abs/1905.11946) (Tan & Le, 2019). The core insight is **compound scaling**: simultaneously scaling network depth, width, and input resolution using a fixed set of coefficients derived via neural architecture search. The baseline architecture (B0) uses Mobile Inverted Bottleneck Convolution (MBConv) blocks with Squeeze-and-Excitation attention.

timm's implementation (`timm/models/efficientnet.py`) covers a wide family of related efficient architectures through the shared `EfficientNet` model class.

## Model Variants

<Tabs>
  <Tab title="EfficientNet B0–B8">
    Standard EfficientNet family scaled from B0 (smallest) to B8 (largest). L2 is an even larger variant trained with NoisyStudent.

    | Model | Input Size | Top-1 (IN-1k) | Params |
    |-------|-----------|--------------|--------|
    | `efficientnet_b0` | 224 | ~77.7% | 5.3M |
    | `efficientnet_b1` | 240 | ~79.2% | 7.8M |
    | `efficientnet_b2` | 260 | ~80.4% | 9.2M |
    | `efficientnet_b3` | 300 | ~81.7% | 12M |
    | `efficientnet_b4` | 380 | ~83.0% | 19M |
    | `efficientnet_b5` | 456 | ~83.7% | 30M |
    | `efficientnet_b6` | 528 | ~84.1% | 43M |
    | `efficientnet_b7` | 600 | ~84.4% | 66M |
    | `efficientnet_b8` | 672 | ~85.5% | 88M |
    | `efficientnet_l2` | 800 | ~88.3% (NoisyStudent) | 480M |

    TensorFlow-ported variants are also available with the `tf_` prefix (e.g., `tf_efficientnet_b0` through `tf_efficientnet_b8`), which use slightly different preprocessing.
  </Tab>
  <Tab title="EfficientNet-V2">
    [EfficientNetV2: Smaller Models and Faster Training](https://arxiv.org/abs/2104.00298) introduces Fused-MBConv blocks in early stages and progressive training for faster throughput:

    | Model | Input Size | Params |
    |-------|-----------|--------|
    | `efficientnetv2_rw_t` | 224 | 13.7M |
    | `efficientnetv2_rw_s` | 384 | 23.9M |
    | `efficientnetv2_rw_m` | 416 | 53.2M |
    | `efficientnetv2_s` | 384 | 21.5M |
    | `efficientnetv2_m` | 480 | 54.1M |
    | `efficientnetv2_l` | 480 | 119M |
    | `efficientnetv2_xl` | 512 | 208M |

    `rw_` variants are timm's own reimplementations with slightly different hyperparameters than the official models.
  </Tab>
  <Tab title="EfficientNet-Lite & Edge">
    Variants optimized for specific deployment targets:

    **EfficientNet-Lite** (no SE, ReLU6 activation — hardware-friendly):
    - `efficientnet_lite0`, `efficientnet_lite1`, `efficientnet_lite2`, `efficientnet_lite3`, `efficientnet_lite4`

    **EfficientNet-EdgeTPU** (optimized for Google Edge TPU):
    - `efficientnet_es` (Small), `efficientnet_em` (Medium), `efficientnet_el` (Large)
    - TF-ported: `tf_efficientnet_es`, `tf_efficientnet_em`, `tf_efficientnet_el`

    **EfficientNet-CondConv** (conditionally parameterized convolutions):
    - `efficientnet_cc_b0_4e`, `efficientnet_cc_b0_8e`, `efficientnet_cc_b1_8e`
  </Tab>
  <Tab title="MixNet">
    [MixConv: Mixed Depthwise Convolutional Kernels](https://arxiv.org/abs/1907.09595) replaces standard depthwise convolutions with a mixture of kernel sizes in a single layer:

    | Model | Params |
    |-------|--------|
    | `mixnet_s` | 4.1M |
    | `mixnet_m` | 5.0M |
    | `mixnet_l` | 7.3M |
    | `mixnet_xl` | 11.9M |
    | `mixnet_xxl` | — |

    MixNet models share the `EfficientNet` class and are built with the same block infrastructure.
  </Tab>
  <Tab title="MNasNet & Others">
    Additional architectures sharing the EfficientNet infrastructure:

    **MNasNet** ([MnasNet: Platform-Aware NAS for Mobile](https://arxiv.org/abs/1807.11626)):
    - `mnasnet_050`, `mnasnet_075`, `mnasnet_100`, `mnasnet_140`, `mnasnet_small`

    **MobileNetV2** (via EfficientNet builder):
    - `mobilenetv2_035`, `mobilenetv2_050`, `mobilenetv2_075`, `mobilenetv2_100`, `mobilenetv2_140`, `mobilenetv2_110d`, `mobilenetv2_120d`

    **TinyNet** ([Model Rubik's Cube](https://arxiv.org/abs/2010.14819)):
    - `tinynet_a`, `tinynet_b`, `tinynet_c`, `tinynet_d`, `tinynet_e`
  </Tab>
</Tabs>

## Key Parameters

The `EfficientNet` class constructor accepts these key arguments:

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `num_classes` | `int` | `1000` | Number of output classes. |
| `in_chans` | `int` | `3` | Number of input image channels. |
| `drop_rate` | `float` | `0.0` | Classifier dropout rate. |
| `drop_path_rate` | `float` | `0.0` | Stochastic depth rate. |
| `global_pool` | `str` | `'avg'` | Global pooling type for the head. |
| `num_features` | `int` | `1280` | Output channels of the final conv before pooling. |
| `act_layer` | layer | SiLU | Activation function. |
| `norm_layer` | layer | BatchNorm2d | Normalization layer. |

## Code Examples

<AccordionGroup>
  <Accordion title="Load EfficientNet-B0 with pretrained weights">
    ```python
    import timm
    import torch

    model = timm.create_model('efficientnet_b0', pretrained=True)
    model.eval()

    x = torch.randn(1, 3, 224, 224)
    with torch.no_grad():
        output = model(x)

    print(output.shape)  # torch.Size([1, 1000])
    ```
  </Accordion>
  <Accordion title="EfficientNet-V2 for transfer learning">
    ```python
    import timm

    # Load EfficientNetV2-S, replace head for 10 classes
    model = timm.create_model(
        'efficientnetv2_s',
        pretrained=True,
        num_classes=10,
    )

    # Get the data config expected by this model's pretrained weights
    data_config = timm.data.resolve_model_data_config(model)
    print(data_config)
    # {'input_size': (3, 384, 384), 'interpolation': 'bicubic', ...}
    ```
  </Accordion>
  <Accordion title="Extract feature maps">
    ```python
    import timm
    import torch

    # Create a feature extractor (removes the classifier head)
    model = timm.create_model(
        'efficientnet_b3',
        pretrained=True,
        features_only=True,
    )
    model.eval()

    x = torch.randn(1, 3, 300, 300)
    with torch.no_grad():
        features = model(x)

    # Returns list of feature maps at each stage
    for i, f in enumerate(features):
        print(f'Stage {i}: {f.shape}')
    # Stage 0: torch.Size([1, 24, 150, 150])
    # Stage 1: torch.Size([1, 32, 75, 75])
    # Stage 2: torch.Size([1, 48, 38, 38])
    # Stage 3: torch.Size([1, 136, 19, 19])
    # Stage 4: torch.Size([1, 384, 10, 10])
    ```
  </Accordion>
  <Accordion title="MixNet example">

<Tip>
Use `timm.data.resolve_model_data_config(model)` to get the correct input size, normalization mean/std, and interpolation mode expected by each pretrained model. EfficientNet variants have different default resolutions (B0=224, B1=240, B2=260, etc.).
</Tip>
