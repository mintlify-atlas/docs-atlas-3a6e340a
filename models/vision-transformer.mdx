title: "Vision Transformer (ViT)"
description: "Pure transformer architecture for image recognition. timm includes ViT-Tiny through ViT-Huge with standard, CLIP, DINOv2, and register-token pretrained weights."
---

Vision Transformer (ViT) was introduced in [An Image is Worth 16x16 Words: Transformers for Image Recognition at Scale](https://arxiv.org/abs/2010.11929). It splits an image into fixed-size non-overlapping patches, linearly embeds each patch, prepends a learnable `[CLS]` token, adds position embeddings, and feeds the resulting sequence through a standard Transformer encoder.

timm's implementation (`timm/models/vision_transformer.py`) supports a wide range of variants including standard ImageNet-pretrained models, CLIP-pretrained models, DINOv2 models, and models with register tokens.

## Key Parameters

The `VisionTransformer` class accepts the following constructor arguments (all can be passed via `create_model` kwargs):

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `img_size` | `int \| tuple` | `224` | Input image size. |
| `patch_size` | `int \| tuple` | `16` | Size of each image patch. |
| `in_chans` | `int` | `3` | Number of input channels. |
| `num_classes` | `int` | `1000` | Number of output classes. Set to `0` to get features only. |
| `global_pool` | `str` | `'token'` | Pooling mode: `'token'`, `'avg'`, `'avgmax'`, `'map'`, or `''`. |
| `embed_dim` | `int` | `768` | Transformer embedding dimension. |
| `depth` | `int` | `12` | Number of transformer blocks. |
| `num_heads` | `int` | `12` | Number of attention heads. |
| `mlp_ratio` | `float` | `4.0` | MLP hidden dim ratio relative to embed dim. |
| `reg_tokens` | `int` | `0` | Number of register tokens to prepend. |
| `drop_rate` | `float` | `0.0` | Classifier head dropout rate. |
| `drop_path_rate` | `float` | `0.0` | Stochastic depth rate. |
| `dynamic_img_size` | `bool` | `False` | Allow variable input sizes at inference. |

<Note>
`global_pool='token'` uses the `[CLS]` token output for classification (original ViT behavior). `global_pool='avg'` averages all patch tokens, which often improves performance on fine-tuned models.
</Note>

## Code Examples

<AccordionGroup>
  <Accordion title="Basic loading and inference">
    ```python
    import timm
    import torch

    # Load a pretrained ViT-Base/16 model
    model = timm.create_model('vit_base_patch16_224', pretrained=True)
    model.eval()

    # Create a random input (batch=1, channels=3, height=224, width=224)
    x = torch.randn(1, 3, 224, 224)

    with torch.no_grad():
        output = model(x)   # shape: (1, 1000)

    print(output.shape)  # torch.Size([1, 1000])
    ```
  </Accordion>
  <Accordion title="Extract intermediate features">
    ```python
    import timm
    import torch

    model = timm.create_model('vit_base_patch16_224', pretrained=True)
    model.eval()

    x = torch.randn(1, 3, 224, 224)

    with torch.no_grad():
        # forward_features returns the full sequence before the classifier head
        features = model.forward_features(x)

    # Shape: (batch, num_patches + 1, embed_dim)
    # For vit_base_patch16_224: (1, 197, 768)
    # 197 = 14*14 patches + 1 CLS token
    print(features.shape)
    ```
  </Accordion>
  <Accordion title="Fine-tuning with custom classes">
    ```python
    import timm

    # Load pretrained backbone, replace head for 10-class task
    model = timm.create_model(
        'vit_small_patch16_224',
        pretrained=True,
        num_classes=10,
    )

    # Or reset the head after loading
    model = timm.create_model('vit_small_patch16_224', pretrained=True)
    model.reset_classifier(num_classes=10)
    ```
  </Accordion>
  <Accordion title="DINOv2 with register tokens">
    ```python
    import timm
    import torch

    # Load DINOv2 ViT-B/14 with 4 register tokens
    model = timm.create_model(

    ```
  </Accordion>

    import timm

## Register Tokens

<Note>
Register tokens were introduced in [Vision Transformers Need Registers](https://arxiv.org/abs/2309.13549) (Meta AI, 2023). They are extra learnable tokens that attend freely to all patches, absorbing "artifact" attention patterns that otherwise appear in high-norm patch tokens. This leads to cleaner attention maps and improved dense prediction performance.
</Note>

Variants with `_reg4_` in the name include 4 register tokens. The `reg_tokens` parameter controls this at model construction time:

```python
import timm

# Manually create a ViT with register tokens
model = timm.create_model(
    'vit_base_patch16_224',
    pretrained=False,
    reg_tokens=4,
)
```

When `reg_tokens > 0`, `forward_features` returns shape `(B, num_patches + 1 + reg_tokens, embed_dim)`.
